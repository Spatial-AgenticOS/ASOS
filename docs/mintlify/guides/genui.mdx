# Gen-UI in FERAL

Gen-UI is server-driven UI (SDUI): the brain emits structured UI payloads, the web client renders supported components.

## What is shipping now

- SDUI payload rendering in the web UI (`SduiRenderer` path).
- SDUI patch updates (`sdui_patch`) for incremental UI changes.
- App surface routing and sandboxed rendering flows.
- Live frame inspection in `/canvas`.

## What this is not

- Not arbitrary HTML generation.
- Not full native SDUI parity across every client surface.
- Not a claim that all marketplace trust/signing flows are complete end-to-end.

## Core shape

Example SDUI payload:

```json
{
  "type": "sdui_payload",
  "payload": {
    "component": "MetricCard",
    "props": {
      "title": "CPU temperature",
      "value": "72C"
    }
  }
}
```

The brain decides component + props; the client renders known component contracts.

## Where to look

- Brain-side generation/contracts: `feral-core/genui/`
- Runtime routing and app registry: `feral-core/agents/`
- Client renderer: `feral-client-v2/src/ui/SduiRenderer.jsx`
- Live inspector: `feral-client-v2/src/pages/GenUICanvas.jsx`

## Practical guidance

- Use Gen-UI for structured operational surfaces: cards, forms, metrics, stateful action flows.
- Keep component contracts explicit and versioned.
- Treat unsupported component types as fallback-only until renderer support is shipped.
